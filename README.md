# ECHO Ransomware UI & Persistence Simulation (Demo)

ECHO is a specialized, Python-based Proof-of-Concept (PoC) graphical simulation designed to model the behavior, user interface tactics, and persistence mechanisms commonly deployed by modern screen-locking malware variants.

## 🔬 Architectural Overview

The ECHO Demo simulates the precise high-stress environments generated by advanced persistent threats (APTs), allowing security analysts to observe and monitor localized system impacts safely within sandbox environments.

### Key Capabilities & Mechanics

* **Aggressive UI Control:** Utilizes Tkinter to implement a persistent, topmost, borderless full-screen layout that intercepts standard window-close triggers (`WM_DELETE_WINDOW`) to simulate system lock-out state.
* **Asynchronous Multi-Threading:** Offloads resource-heavy operational tasks—such as external geolocation lookup (via IP-API) and standard webcam initialization (via OpenCV)—to concurrent background threads, ensuring an uninterrupted rendering loop.
* **Process Termination Loop:** Executes an active background worker thread that continuously calls `taskkill` against `Taskmgr.exe` every 500ms, mimicking anti-analysis and defense evasion techniques.
* **Low-Level Persistence Modeling:** Provisions a stealth launch wrapper (`system_service.vbs`) within the user's local Startup folder (`%APPDATA%\Microsoft\Windows\Start Menu\Programs\Startup`) to demonstrate basic administrative persistence vectors.
* **Psychological Exploitation Vectors:** Implements interactive UI stress elements including dynamic Canvas-based Matrix digital rain animations, localized window-shaking sequences on erroneous inputs, random low-frequency auditory alerts via the Windows sound subsystem, and multi-threaded text-to-speech (TTS) warning alerts.
